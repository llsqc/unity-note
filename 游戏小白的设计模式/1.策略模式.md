别怕，即使你完全没接触过设计模式，只要跟着我的思路走，保证你能在 10 分钟内彻底弄懂它，还能马上运用到你的 Unity 项目里！

### 1. 通俗易懂的例子（生活隐喻）

想象你要**从北京去上海**。

❌ **错误的做法（不使用策略模式）：**
你的大脑里有一个死板的“出行方法”。每次出门，你脑子里的细胞都要跑一遍极长的判断流程：
“如果今天有钱，我就坐飞机去；如果钱不多，我就坐高铁去；如果想锻炼身体，我就骑自行车去；如果是世界末日，我就走路去……”
万一哪天新开通了“超级磁悬浮列车”，你还得给自己做个“开颅手术”（修改内部代码），在脑子里再加一条“如果……”的判断。这太累了，而且极容易改错，导致你连路都不会走了。

✅ **正确的做法（使用策略模式）：**
你不再去死记硬背各种出行方式的具体步骤。你只是拿着一个 **“交通工具遥控器”（接口）**。
你花钱雇了几个专业的司机：“飞机驾驶员”、“高铁司机”、“共享单车大爷”**（具体的策略类）**。
出门时，你只要把其中一个司机绑定到你的遥控器上，然后按下 **“出发”** 按钮。
想换交通工具？直接换个司机就行。明天发明了“传送门”？新招一个“传送门操作员”即可，你的大脑（核心代码）完全不需要动手术！

**核心思想**：把“怎么做”的具体步骤封装起来，让你随时随地可以“偷天换日”地替换它们，而不影响调用它的人。

### 2. Unity 游戏实战场景

🎮 **典型场景：主角的武器攻击系统（Weapon Attack System）**

**为什么适合？**
在动作游戏里，玩家经常要在“大剑”、“弓箭”、“魔法法杖”之间来回切换。
如果不使用策略模式，你的 `Player` 脚本里的 `Attack()` 方法会充斥着巨量的 `if...else` 或 `switch...case`：
```csharp
if (weapon == "Sword") { /* 播放挥砍动画、做近战射线检测... */ }
else if (weapon == "Bow") { /* 实例化弓箭预制体、计算抛物线... */ }
else if (weapon == "Magic") { /* 消耗蓝量、生成AOE范围伤害... */ }
```
这样写不仅代码长达几千行，而且未来你想加一个“平底锅”武器，就必须打开 `Player` 脚本去修改，极容易把原本正常的“大剑”逻辑给改坏了（违反了开闭原则）。

使用策略模式，主角只需拥有一个 `IAttackStrategy`（攻击策略），按下攻击键时，调用策略的攻击方法即可。随时换武器，随时换策略！

### 3. C# 代码实现（分步讲解）

为了让策略更加轻量化，我们的策略类不需要继承 `MonoBehaviour`，它们只是普通的 C# 类。

#### 第一步：定义“交通工具遥控器”（攻击策略接口）
我们先定义一个标准，所有的武器都必须遵守这个标准。
```csharp
// ⚔️ 攻击策略接口
public interface IAttackStrategy
{
    // 所有具体的攻击策略，都必须实现这个方法
    void Attack();
}
```

#### 第二步：编写具体的实现类（各种武器的攻击方式）
现在我们来写具体的“司机”。它们独立运作，互不干扰。

```csharp
using UnityEngine;

// 🗡️ 具体策略A：大剑近战攻击
public class MeleeAttackStrategy : IAttackStrategy
{
    public void Attack()
    {
        // 具体的近战逻辑：播放挥砍动画、检测前方扇形区域的敌人等
        Debug.Log("🗡️ 挥舞大剑：造成了 50 点近战物理伤害！");
    }
}

// 🏹 具体策略B：弓箭远程攻击
public class RangedAttackStrategy : IAttackStrategy
{
    public void Attack()
    {
        // 具体的远程逻辑：生成箭矢、赋予向前的刚体力等
        Debug.Log("🏹 射出弓箭：造成了 30 点远程穿刺伤害！");
    }
}

// ✨ 具体策略C：法杖魔法攻击
public class MagicAttackStrategy : IAttackStrategy
{
    public void Attack()
    {
        // 具体的魔法逻辑：消耗MP，在目标脚下生成火墙等
        Debug.Log("✨ 吟唱魔法：消耗 10 点 MP，造成了 100 点范围火焰伤害！");
    }
}
```

### 4. 在 Unity 中如何使用？

现在，我们回到主角身上。主角不再需要知道大剑和魔法具体是怎么工作的，他只负责**装备策略**和**使用策略**。

```csharp
using UnityEngine;

// 🦸‍♂️ 客户端：玩家控制器
public class PlayerController : MonoBehaviour
{
    // 这就是我们的“遥控器”，用来存放当前的攻击策略
    private IAttackStrategy _currentAttackStrategy;

    void Start()
    {
        // 游戏刚开始，默认给玩家装备一把大剑
        SetWeapon(new MeleeAttackStrategy());
        Debug.Log("🎮 游戏开始，默认装备：大剑");
    }

    void Update()
    {
        // 🔁 动态切换策略：按下键盘 1, 2, 3 切换武器
        if (Input.GetKeyDown(KeyCode.Alpha1))
        {
            SetWeapon(new MeleeAttackStrategy());
            Debug.Log("🔄 切换武器为：大剑");
        }
        else if (Input.GetKeyDown(KeyCode.Alpha2))
        {
            SetWeapon(new RangedAttackStrategy());
            Debug.Log("🔄 切换武器为：弓箭");
        }
        else if (Input.GetKeyDown(KeyCode.Alpha3))
        {
            SetWeapon(new MagicAttackStrategy());
            Debug.Log("🔄 切换武器为：法杖");
        }

        // ⚔️ 执行策略：按下空格键进行攻击
        if (Input.GetKeyDown(KeyCode.Space))
        {
            PerformAttack();
        }
    }

    // 换武器（换策略）的方法
    public void SetWeapon(IAttackStrategy newStrategy)
    {
        _currentAttackStrategy = newStrategy;
    }

    // 核心：主角根本不管怎么攻击，直接让当前的策略去干活
    private void PerformAttack()
    {
        if (_currentAttackStrategy != null)
        {
            _currentAttackStrategy.Attack();
        }
    }
}
```

**📢 预期的控制台 (Debug.Log) 输出流程：**
1. 运行游戏：`🎮 游戏开始，默认装备：大剑`
2. 按下空格键：`🗡️ 挥舞大剑：造成了 50 点近战物理伤害！`
3. 按下数字键 2：`🔄 切换武器为：弓箭`
4. 再次按空格键：`🏹 射出弓箭：造成了 30 点远程穿刺伤害！`
5. 按下数字键 3：`🔄 切换武器为：法杖`
6. 再次按空格键：`✨ 吟唱魔法：消耗 10 点 MP，造成了 100 点范围火焰伤害！`

### 5. 核心原理与优缺点总结

**🎯 大白话总结：**
策略模式解决的**最大痛点**就是 **“消灭无限膨胀的 `if...else`”**。它把不同的“做法”打包成一个个独立的模块，让主角可以在运行时像换衣服一样，随时换上一套新的“做法”。

**✅ 优点：**
1. **符合开闭原则（极其好维护）**：如果策划说明天要加个“激光枪”，你只需要新建一个 `LaserAttackStrategy` 类就行，完全不用改动主角的代码！
2. **极高的灵活性**：在游戏运行期间，随时随地无缝切换行为。
3. **避免代码臃肿**：让主角类的代码变得非常干净，职责单一。

**❌ 缺点：**
1. **类的数量增加**：每个策略都要写一个类。如果有 50 把武器，就有 50 个策略类（虽然这是值得的）。
2. **客户端必须了解所有策略**：玩家（或者 UI、输入系统）必须要知道有哪些策略存在，才能进行切换（比如知道按下 1 是大剑，2 是弓箭）。

**💡 Unity 中的其他常见应用场景：**
1. **伤害计算公式**：普通伤害、暴击伤害、无视护甲的真实伤害等，通过替换策略来计算最终扣血量。
2. **背包排序功能**：按获取时间排序、按品质排序、按价格排序，把排序算法封装成不同的策略。
3. **寻路与移动逻辑**：比如飞行怪物的直线寻路策略，和地面怪物的 NavMesh 避障寻路策略。

**⚠️ 导师防坑提示（区分混淆）：**
很多新手容易把**策略模式（Strategy）** 和**状态模式（State）** 搞混，因为它们俩的代码长得极其相似（都是提取接口，都是委托给内部对象）。
*   **一句话点明核心区别**：**策略模式**是“客户端（玩家）**主动**说我要换方法”；而**状态模式**是“状态在内部**自动**流转（比如怪物血量低于20%自动从‘巡逻状态’切到‘狂暴状态’）”。策略更强调**替换算法**，状态更强调**状态的自我变迁**。
