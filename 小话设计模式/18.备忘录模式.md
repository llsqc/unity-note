你好！我是你的 Unity 开发导师。今天我们要聊的是游戏开发中的“后悔药”——**备忘录模式（Memento Pattern）**。

在游戏里，玩家最怕的就是“一失足成千古恨”。为了给玩家提供更好的体验，我们经常需要实现“存档/读档”或者“撤销（Undo）”功能。备忘录模式就是专门为了**捕获并保存一个对象的内部状态**，以便以后能恢复它，同时又不破坏它的封装性。

---

## 1. 通俗易懂的例子：单机游戏的“快速存档” 💾

想象你正在玩《艾尔登法环》，准备去挑战一个极其凶残的 BOSS。

* **❌ 错误的做法（不使用模式）：**
    你为了记录当前的状态，不得不把你的血量、蓝量、身上的装备、站立的坐标全部手写在一张纸上。等你被打败了，你再照着这张纸，一个一个地把游戏里的数值改回去。
    * **麻烦：** 太累了！而且你作为一个玩家，竟然能直接修改游戏内部的坐标和数值，这极其不安全，很容易把游戏逻辑搞崩溃。

* **✅ 正确的做法（备忘录模式）：**
    你在进雾门前按了一下 **F5（快速存档）**。游戏瞬间产生了一个“存档快照”，并把它锁在一个盒子里交给“存档管理员”。
    * **巧妙之处：** 你（玩家）打不开这个盒子，不知道里面是怎么存的。当你被打败后，按一下 **F9（快速读档）**，管理员把盒子还给游戏系统，游戏瞬间恢复到进雾门前的状态。
    * **结果：** 游戏逻辑依然严密，你也能轻松实现“反复受苦”。

## 2. Unity 游戏实战场景：关卡编辑器的“撤销/重做”系统 🛠️

在 Unity 开发中，如果你在做一个**关卡编辑器**或者**解谜游戏**（比如推箱子），备忘录模式是最完美的方案。

* **为什么要用？**
    在推箱子游戏中，玩家推错一步就可能导致死局。我们需要记录箱子的坐标。
* **应用效果：**
    每当玩家移动一次，我们就生成一个备忘录（Memento），存入一个“历史记录栈”。玩家按下 `Ctrl+Z` 时，我们就弹出栈顶的备忘录，让箱子回到上一个位置。这种方式不需要记录复杂的移动轨迹，只需要还原当时的“快照”。

## 3. C# 代码实现（分步讲解）

### 第一步：定义备忘录类（快照盒子）

```csharp
using UnityEngine;

// 📦 备忘录：纯粹的数据容器，用来存状态
public class PlayerMemento
{
    // 这里的属性通常是只读的，防止外部乱改
    public int Health { get; private set; }
    public Vector3 Position { get; private set; }

    public PlayerMemento(int hp, Vector3 pos)
    {
        Health = hp;
        Position = pos;
    }
}
```

### 第二步：编写发起人（Player - 游戏角色）

```csharp
// 🎮 发起人：也就是需要被保存状态的对象
public class PlayerOriginator : MonoBehaviour
{
    public int Health = 100;

    // 📸 创建备忘录：把当前状态打包
    public PlayerMemento Save()
    {
        Debug.Log($"[玩家] 状态已打包：血量 {Health}, 位置 {transform.position}");
        return new PlayerMemento(Health, transform.position);
    }

    // 🔙 恢复备忘录：从盒子里拿回状态
    public void Restore(PlayerMemento memento)
    {
        Health = memento.Health;
        transform.position = memento.Position;
        Debug.Log($"[玩家] 状态已恢复！当前血量: {Health}");
    }
}
```

### 第三步：编写管理者（Caretaker - 存档管理器）

```csharp
using System.Collections.Generic;

// 🗄️ 管理者：只负责存盒子，不准看盒子里面的内容
public class SaveManager
{
    // 用栈来存，方便实现“撤销”功能（后进先出）
    private Stack<PlayerMemento> _history = new Stack<PlayerMemento>();

    public void PushSave(PlayerMemento memento) => _history.Push(memento);

    public PlayerMemento PopSave()
    {
        if (_history.Count > 0) return _history.Pop();
        return null;
    }
}
```

## 4. 在 Unity 中如何使用？

我们通过一个简单的脚本模拟战斗中的存档与回溯。

```csharp
using UnityEngine;

public class MementoTest : MonoBehaviour
{
    private PlayerOriginator _player;
    private SaveManager _saveManager = new SaveManager();

    void Start()
    {
        _player = gameObject.AddComponent<PlayerOriginator>();

        // 1. 初始状态并存个档
        _player.Health = 100;
        transform.position = Vector3.zero;
        _saveManager.PushSave(_player.Save());

        // 2. 模拟被怪打了，跑了一段距离
        Debug.Log("--- 玩家遇到伏击，状态发生改变 ---");
        _player.Health = 20;
        transform.position = new Vector3(10, 0, 5);

        // 3. 玩家觉得不行，点击“回溯”
        Debug.Log("--- 玩家发动了『时间回溯』 ---");
        PlayerMemento memento = _saveManager.PopSave();
        if (memento != null)
        {
            _player.Restore(memento);
        }
    }
}

/* 预期 Debug.Log 输出结果：
1. [玩家] 状态已打包：血量 100, 位置 (0, 0, 0)
2. --- 玩家遇到伏击，状态发生改变 ---
3. --- 玩家发动了『时间回溯』 ---
4. [玩家] 状态已恢复！当前血量: 100
*/
```

## 5. 核心原理与优缺点总结

### 💡 它解决了什么痛点？
解决了 **“在不破坏封装性的前提下，保存和恢复对象状态”** 的问题。它让对象（发起人）专注于自己的业务逻辑，而不需要暴露内部细节给外部管理者。

### ✅ 优点
1.  **封装性好：** 只有发起人自己能操作备忘录，管理者看不见也改不了里面的数据。
2.  **简化发起人：** 存档的存储和维护由管理者负责，发起人只管“生”和“收”。
3.  **支持多级撤销：** 配合栈（Stack）可以轻松实现无限次的 Undo。

### ❌ 缺点
1.  **内存开销：** 如果对象很大且存档频繁（比如每秒存一次），会占用大量内存。
2.  **管理成本：** 需要额外定义备忘录类和管理者类。

### 🎮 Unity 中的其他常见场景
* **策略游戏中的“悔棋”：** 记录棋盘上所有棋子的状态快照。
* **角色捏脸系统：** 保存每一个滑竿参数的快照，方便玩家对比前后的效果。
* **对话系统的分支回退：** 记录当前已触发的剧情变量，方便跳回之前的剧情分支。

### ⚠️ 导师点拨：它和“原型模式”的区别
一句话点明核心区别：
> **原型模式**是创建了一个**独立的新对象**（复印件），你可以随便玩那个新对象；而**备忘录模式**是保存了一个**旧的状态数据**，目的是为了把**原对象**恢复到过去。
